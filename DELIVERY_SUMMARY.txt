✨ SOLID PRINCIPLES IN ACTION
═══════════════════════════════════════════════════════════════════════════════

✓ SINGLE RESPONSIBILITY
  └─ Each file has ONE clear purpose
     • authApi.ts = Auth API calls only
     • useAuth.ts = React Query setup only
     • LoginForm.tsx = UI rendering only
     • useAuthStore = State management only

✓ OPEN/CLOSED
  └─ Easy to extend without modifying
     • Add new domain following the pattern
     • New endpoints without touching UI
     • New components without changing existing

✓ LISKOV SUBSTITUTION
  └─ Components/hooks are interchangeable
     • Same interface contracts
     • Consistent prop patterns
     • Easy to swap implementations

✓ INTERFACE SEGREGATION
  └─ Small focused interfaces
     • TransferPayload for transfers
     • LoginPayload for login
     • PaymentInitiationPayload for payments
     • Components receive ONLY needed props

✓ DEPENDENCY INVERSION
  └─ Depend on abstractions, not concrete implementations
     • Components → Hooks abstraction
     • Hooks → API Service abstraction
     • Service → API Client abstraction
     • Easy to mock and test


═══════════════════════════════════════════════════════════════════════════════
🔒 AUTHENTICATION & SECURITY
═══════════════════════════════════════════════════════════════════════════════

✓ JWT Token Management
  ├─ Automatic injection in headers
  ├─ Secure localStorage storage
  ├─ Auto-refresh on 401
  └─ Request queuing during refresh

✓ Token Lifecycle
  ├─ On login: Store access + refresh tokens
  ├─ On request: Inject access token
  ├─ On 401: Refresh token automatically
  ├─ On logout: Clear all tokens + state
  └─ On page load: Restore from localStorage
